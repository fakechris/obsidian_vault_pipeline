## 0. 设计原则

1. **产品面 = 用户任务**，不是管线控制台（见 IMPLEMENTATION_PLAN Stage P.4）。
2. **每个页面回答一个用户问题**；回答不了具体问题的页面不存在。
3. **一切皆链接**：任何实体（源/卡片/单元/主张/主题）在任何地方出现都可点击进入其详情。
4. **图谱是组件不是页面**：在哪个上下文就默认显示哪个范围；全局图只是 Knowledge 页的一种视图。
5. **一个前端、一套导航、一套主题 token**。旧的独立 /viz 导航废除。
6. **国际化而非双语并排**（operator 2026-07-08 定案）：默认英文单语界面；设置中切换
   简体中文（整套 UI 文案翻译，非并排）。`localStorage['ovp-lang']`，与主题同机制；
   B1 期间语言开关临时放顶栏（EN/中），B5 落进设置页。mockup 里的"Today 今天"
   并排样式作废。

## 3. 页面系统（6 个一级目的地 + 3 类详情页）

### 3.1 今天 Today `/`
回答 US1/US6。
```
┌────────┬──────────────────────────────────────────────┐
│ 侧边栏  │ 今天 Today                       ◐主题 ●状态灯 │
│        │ ┌────────────────────────────────────────┐   │
│ ●今天   │ │ 今日摘要 digest（有则显示，无则收起）      │   │
│  资料   │ └────────────────────────────────────────┘   │
│  搜索   │ [进来 3] [读完 8] [新结晶 2] [待处理 1]        │← 当日增量，点击进对应页
│  知识   │ 需要你 Attention（仅在非空时出现）             │
│  对话   │ ┃ blocked · <源标题> · 为什么 · [处理→资料详情] │
│  系统   │ 时间线 Timeline（今天→往前 7 天，紧凑行）       │
│        │  07-08  读完 8 源 · 结晶 +2 durable  [明细]    │
└────────┴──────────────────────────────────────────────┘
```
跳转：统计卡→对应列表页（带筛选参数）；attention→资料详情；时间线行→当日 run 明细（系统页锚点）。

### 3.2 资料 Library `/library` + 源详情 `/library/:sha`
回答 US2/US6。资料页有**页内二级导航**（集合 × 时间 × 状态三个维度），不是一张平铺大表：

```
┌──────────────┬────────────────────────────────────────────┐
│ 集合 Collections│  [列表 ⇄ 卡片]  状态筛选: 全部|待读|已处理|阻塞  │
│ ● 全部 105    │  ┌─ 2026-07 ────────────────────────────┐  │
│   剪藏 Clippings│  │ pill 标题            日期   备注      │  │
│   书签 Pinboard │  │ …按月分组，新月在前，组头可折叠…        │  │
│   捕获 Capture  │  └─ 2026-06 ────────────────────────────┘  │
│ 按月 By month  │                                            │
│   2026-07 (31)│   URL 参数化：/library?c=pinboard&m=2026-07 │
│   2026-06 (58)│                                            │
└──────────────┴────────────────────────────────────────────┘
```

**每个集合的列和形态不同**（同一 EntityRow 组件的变体）：
- **剪藏 Clippings**：标题、日期、读取状态、卡片/单元数（已读完的显示记忆密度）。
- **书签 Pinboard**：URL 域名 + favicon、同步日期、**内容状态**（bare bookmark 缺正文 →
  needs-content 徽标 + "抓取正文"指引；已 enrich 的和剪藏同权）。书签是 URL 中心的，
  默认卡片视图更合适。
- **捕获 Capture**（00-Capture 手动放入）：同剪藏。

列表 = 全量资产（月分组、集合过滤、状态筛选、失败备注），全部状态可 URL 参数化直达。
**源详情 = 三层钻取页**：
```
┌ 面包屑：资料 / <标题>                                    ┐
│ 标题 + 状态 pill + url + 日期 + 位置 + 最近运行            │
│ ┌──────────────────────────────┬─────────────────────┐ │
│ │ Tab: 记忆 Memory | 原文 Source │ 关联图谱（组件，       │ │
│ │  卡片列表（标题+正文）          │  scope=本源邻域：      │ │
│ │  接地单元（引文+行号，          │  本源→卡片→引用它      │ │
│ │  点击行号跳原文 tab 对应位置）   │  的主张→兄弟源）       │ │
│ └──────────────────────────────┴─────────────────────┘ │
│ 原文 tab = 真正的 md 阅读视图：完整 markdown 渲染（标题层级/  │
│ 代码块/引用/图片）、阅读排版（约 720px 行宽）、行号锚点（单元  │
│ 引用跳转目标）、[展开全宽] 阅读模式；不是转义后的纯文本。      │
│ 结晶 Claims 引用本源的主张（每条→主张详情）                 │
└─────────────────────────────────────────────────────────┘
```

### 3.3 知识 Knowledge `/knowledge` + 主题详情 `/knowledge/theme/:t` + 主张详情 `/knowledge/claim/:id`
回答 US3/US4/US7。
- **知识首页**：主题卡片墙（每主题：claim 数、durable 占比、最近更新）+ 视图切换
  `[列表 ⇄ 图谱]`——图谱视图 = KnowledgeGraph 组件全局 scope（原 /viz Graph 收编于此）。
- **主题详情**：该主题 claims 列表（durable 前、caveated 标注）+ 右侧图谱组件（scope=主题）。
- **主张详情**（US4 的核心页）：
```
┌ 面包屑：知识 / <主题> / claim                             ┐
│ 主张全文 + durable/caveated pill + strength               │
│ 证据 Evidence：每条 citation = 卡片摘要 + 单元引文（行号）   │
│   → 点击进源详情对应位置                                   │
│ 局部图谱（组件，scope=本主张邻域：引用源 + 同主题相邻主张）    │
│ [caveated 时] 复核区：复核历史 + 进入复核流（Phase B5）      │
└──────────────────────────────────────────────────────────┘
```

### 3.4 搜索 Search `/search`（+ 全局 ⌘K 随处唤起）
回答 US2。一个输入框搜全部实体，结果按类型分组（源/卡片/单元/主张/主题），
每条结果 = 状态 pill + 高亮摘要 + 直达链接。顶栏放大镜 / ⌘K 在任何页面唤起浮层，同一组件。

### 3.5 对话 Ask `/ask`
回答 US5。对话流 + 右侧引用面板：回答中的引用编号 [1][2] 悬停出摘要、点击在右栏定位，
右栏每条证据→主张/源详情。历史会话（.ovp/chats）在左侧列表。回答未通过引用校验时明确标注。

### 3.6 系统 System `/system`
回答 US8。运行记录（runs 表 + Flow/Monitor 面板收编）、blocked/失败源、doctor 结果、
概念说明（三层模型：原文→记忆→结晶，durable vs caveated，gate 是什么）、
设置（v1 只读：vault 路径、LLM client、caps、语言、schema 版本）、管理纵深链接（ops/audit/candidates）。

## 4. KnowledgeGraph 组件（KMEM 模式）

一个组件三种 scope，同一渲染引擎（G6，沿用 M33 的社区/密度分层能力）：

| scope | 默认可见范围 | 出现在 |
|---|---|---|
| `global` | 全部主张 + 主题社区 | 知识首页（图谱视图） |
| `theme(t)` | 该主题的主张 + 引用源 | 主题详情右栏 |
| `neighborhood(id, depth≤2)` | 该实体的邻域 | 源详情右栏、主张详情下方 |

交互统一：点节点→侧栏摘要卡（不离开当前页）；双击→跳转该实体详情页；
scope 内搜索框；嵌入态默认高度 ~360px，可展开全屏。**不再有独立的 /viz 路由与导航。**

## 5. 导航模型

- **侧边栏** = 6 个一级目的地（今天/资料/搜索/知识/对话/系统），当前页高亮；折叠态只剩图标。
- **顶栏**（全站一致）：面包屑（详情页）、⌘K 搜索、主题切换（☀/☾）、状态灯（点击→系统）。
- **链接规则**：实体在任何列表/图谱/引用中出现 → 点击进详情；详情页之间靠实体链接横向跳，
  不依赖返回按钮；所有列表页支持 URL 参数化筛选（可分享/可收藏）。

## 6. 视觉基础 = OVP Design System（operator 2026-07-08 提供，定案采纳）

来源：`OVP Design System(1).zip`（operator 交付；工作副本在 `.run/design-system-ref/`，
建造时随 B1 落库到 `console-ui/design-system/`）。**替代此前临时的 M28 token。**

采纳的核心（原样使用，不再发明）：
- **双主题平权**：浅色 "Atelier"（暖羊皮纸 `#f7f6f2` + 赤陶土 `#9f4f24`）/
  深色 "Vault"（近黑 `#0a0e16` + 深蓝 `#3b82f6` + 青 `#06b6d4`）；
  `data-theme` 属性 + `localStorage['ovp-theme']`，初始跟随系统偏好。
- **字体**：IBM Plex Sans + Plex Sans SC（中文）+ Plex Mono，自托管 woff2。
- **图谱 token 已内置**：`--graph-bg/--graph-link/--graph-hull/--c-1…8`（社区色两主题各一套
  调校）——viz 与壳共享同一 data-theme，**风格割裂问题在 token 层解决**；G6 调色全部取自这套变量。
- **气质规则**：quiet utility——1px 边框、radius 阶（10/12/16/20/999）、唯一投影只在外壳、
  零渐变零毛玻璃、text-first 零图标（unicode 箭头），hover 只变下划线/透明度。
- **词汇分层（BL-051 精神）**：用户面用产品词（资料/记忆/主张/主题），内部词（pack/crystal/
  unit/cassette）只出现在系统页与 CLI。

Portal v2 的四个扩展（唯一的新增设计，均从 DS 内部素材派生）：
1. **壳与导航**：采纳 DS 原生的顶栏壳（居中单列 max-width 1180 + 顶部导航词条 + 圆角外壳），
   **放弃此前设计的左侧边栏**——6 个一级目的地放顶栏（今天·资料·搜索·知识·对话·系统），
   与 DS kit 的 Reader 壳同构。⚠ 这是对 §5 导航模型的修订。
2. **语义状态色**：DS 原则是"状态=accent pill+字面词"，但本产品有真语义
   （durable/caveated/blocked）——从 DS 社区色板派生：durable=`--c-3` 苔绿、
   caveated=`--c-4` 金赭、blocked=`--c-6` 谷仓红（两主题各取对应版本），记录为 DS 扩展。
3. **证据密度组件**：EvidenceCard（引文+行号锚）、ClaimCard（状态+strength+引用）按 DS
   卡片规则新绘（16px radius、1px border、无投影）。
4. **"这是什么页？"说明块**：DS kit 的 What-is-this-page 折叠块保留为每页标配
   （承担概念教育，替代独立 Concepts 页的一部分）。
