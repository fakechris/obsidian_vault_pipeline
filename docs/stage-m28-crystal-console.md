## UI structure

Seven surfaces, one shared chrome (left nav + global stat strip). Same information
architecture KnowledgeMem uses, OVP's own truth model underneath.

1. **Attention** (default) — operational review feed, highest-value first. Typed cards:
   Durable Crystal Summary, KMEM Comparison Note, Coverage Gap, High-Value Expansion Candidate,
   Reader-Only Source Needs Partner, Caveated Claim Needs Review, Gate-Protected Item. Each card
   carries a type label, EN + ZH summary, a distinct status pill, why-it-matters, related
   sources, an evidence link, and non-mutating action buttons (View evidence, Find partner
   source, Split / narrow claim, Keep caveated, Dismiss).
2. **Crystal** — 12 durable claims, each with title, body, ZH summary, why-it-matters, theme,
   source count, source links, and an expandable provenance block (claim → cited unit →
   verbatim quote → source line). Header states what *durable* means.
3. **Sources** — all 20 sources: id, title/category, status, OVP card count, KMEM memory count,
   durable claim count, caveated count, durable-claim links, and a link to the article verdict.
4. **Backlog** — `content-backlog.json` rendered: by-type count table + every item with type,
   priority, bilingual summary, related sources/caveated claims, status, recommended action,
   rationale.
5. **Compare** — article-level OVP vs KMEM. Per article: OVP and KMEM core-point coverage,
   factual-issue counts, verdict, bilingual rationale, coverage note, link to the source row. No
   single-memory microscope.
6. **Coverage** — macro view: core-point coverage bars (OVP vs KMEM), source coverage
   (durable / needs-partner / uncovered), theme coverage (durable vs caveated), reader-only /
   unpaired list. No graph visualization, by design.
7. **About** — the OVP pipeline, the durable / caveated / reader-only state model, KMEM as
   reference-only, source article as ground truth, and why Referent / RAG / Graph are
   intentionally kept out of the main path.

### Status model (visually distinct pills)

`durable` (green) · `caveated` (amber) · `reader-only` (blue) · `needs-partner` (violet) ·
`blocked` (red) · `ready-for-review` (teal) · `uncovered` (gray).

A source is **durable** if cited by ≥1 durable claim, **needs-partner** if it only yields
caveated candidates, **uncovered** if it has no claim at all.

## Backlog

`content-backlog.json` — 25 items, derived deterministically. Each item: `id`, `type`,
`priority`, `title`, `summary_en`, `summary_zh`, `related_sources`, `related_caveated`,
`status`, `recommended_action`, `rationale`.

| Type | high | medium | low | total |
|---|---|---|---|---|
| `source_uncovered` | 2 | 0 | 0 | 2 |
| `reader_only_high_value` | 1 | 4 | 0 | 5 |
| `needs_cross_source_partner` | 0 | 11 | 0 | 11 |
| `needs_rewrite_or_split` | 0 | 1 | 0 | 1 |
| `gate_blocked` | 0 | 0 | 2 | 2 |
| `theme_gap` | 0 | 4 | 0 | 4 |
| **total** | **3** | **20** | **2** | **25** |

Routing logic:

- `source_uncovered` — reader-only source with zero durable **and** zero caveated material
  (m18-03, m18-13).
- `reader_only_high_value` — reader-only source that already yields caveated candidates (proof
  durable material exists); high when ≥2 caveated or ≥15 cards.
- caveated claim, `strength == supported`, single source → `needs_cross_source_partner`.
- caveated claim, `over_synthesized` → `needs_rewrite_or_split` (split the fused claim).
- caveated claim, `opinion_as_fact` → `gate_blocked` (held by the strength gate; needs
  independent evidence, not just a rewrite).
- `theme_gap` — caveated theme clusters with no durable anchor.

## KnowledgeMem UI patterns

KMEM was **not** inspected again for this milestone; the runtime, password, and any dump are
untouched. The direction was already derived from prior live observation.

**Adopted (information architecture only):**

- Default Timeline / Attention feed as the home surface
- Typed cards with clear status labels
- Source links on every item
- Review actions (confirm / review / dismiss as non-mutating affordances)
- Crystal items shown as confirmable / reviewable cards
- Separate surfaces for crystal, sources, backlog, compare, coverage

**Rejected:**

- Graph-first / graph-visualization UI
- KMEM's visual skin
- Single-memory microscope as the default review unit
- The memory model itself (KMEM remains reference-only)
